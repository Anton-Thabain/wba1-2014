Positionierung von Elementen

  • static (default)
  • relative
  • absolute
  • fixed
  • (sticky)

position: static

  • Default für alle Elemente
  • top, right, bottom, left, z-index ohne Auswirkungen
  • Inhalte erscheinen in der Reihenfolge, wie sie im Quelltext stehen
.blaue_box {
            position: static;
            bottom: 20px; /*Das würde nichts bewirken*/
            left: 15%;    /*Auch überflüssig*/
        }

position: relative

  • Grundsätzlich wie static
  • Versatzoptionen top, right, bottom, left, z-index sind verfügbar
  • Inhalte erscheinen in der Reihenfolge, wie sie im Quelltext stehen
.blaue_box {
            position: relative;
            bottom: 20px;
            left: 15%;
        }

position: absolute

  • Element wird aus dem normal flow entfernt
  • Versatzoptionen top, right, bottom, left, z-index sind verfügbar
.blaue_box {
            position: absolute;
            bottom: 20px;
            left: 15%;
        }

position: fixed

  • Element wird aus dem normal flow entfernt
  • Versatzoptionen top, right, bottom, left, z-index sind verfügbar
  • Koordinatensystem jedoch immer gemessen am "Viewport" (Browserfenster, Bildschirm mobiler Geräte)
  • Beispiel
.header {
            position: fixed;
            top: 0px;
        }

position: sticky

  • Element wird aus dem normal flow entfernt
  • Versatzoptionen top, right, bottom, left, z-index sind verfügbar
  • Bleibt solange kleben bis der übergeordnete Container aus dem Viewport ist
  • Beispiel (im FF oder Safari anschauen)
.header {
            position: sticky;
            top: 10px;
        }
Fragen zur Positionierung?

Der z-Index

Was wenn Elemente übereinander liegen?

  • Zuletz genanntes Element im HTML Code liegt vorne

.rot {
    z-index: 2;
}

.blau {
    z-index: 3;
}

.gruen {
    z-index: 4;
}
  • Element mit höherem z-index überdeckt Elemente mit niedrigerem z-index

Dokumentenfluss

  • CSS Eigenschaften float und clear
  • CSS Eigenschaften float und clear
  • Float positioniert die Elemente im normal Flow, aber sc
  • float hat die möglichen Eigenschaften left, right, none
  • clear hat die möglichen Eigenschaften left, right, both
  • Problem: Wenn alle Elemente in einem Container floaten, hat dieser die Höhe 0

Lösungen für das Collapse Problem

  • Die Empty Div Method: Am Ende der Box ein <div style="clear: both;">
  • Die Overflow Method: Übergeordnete Container bekommt overflow:show/auto
  • Die Pseudo Element Method: clear:both in einem :after Element, verstehen wir später genauer, was das ist.
