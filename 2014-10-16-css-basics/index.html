Einbindung über eine externe Datei

mit link-Element

<!doctype html>
<html>
  <head>
    <meta charset="utf-8">
    <title>Stylesheet über externe Datei einbinden</title>
    <link rel="stylesheet" href="stylesheet.css" type="text/css">
    <link rel="stylesheet" type="text/css" href="../css/reveal.css">
  </head>
  <body>
  </body>
</html>
  • Stylesheet-Anweisungen werden zentral an einem Ort definiert
  • Stylesheet kann von mehreren Dokumenten eingebunden werden
  • Es können mehrere Stylsheets in einem Dokument eingebunden werden

mit @import

oder in einer CSS-Datei

anderen Regeln stehen.

Einfache Selektoren

* {…} Universalselektor Alle Elemente in einem Dokument
p {…} Typselektor Alle p-Elemente im Dokument
.info {…} Klassenselektor Alle Elemente die der Klasse "Info" angehören
#wichtig {…} ID-Selektor Das Element mit der ID "wichtig"
Eine bestimmte Klasse darf beliebig vielen Elementen in einem Dokument zugewiesen werden. Eine bestimmte ID darf nur einem einzigen Element in einem Dokument zugewiesen werden.

Einfache Selektoren

Einfache Selektoren können auch miteinander kombiniert werden.

p, div {…} Alle p- und div-Elemente
p.info {…} Alle p-Elemente mit der Klasse "info"
.info, .wichtig {…}   Alle Elmente mit der Klasse "info" oder"wichtig"
.info.wichtig {…} Alle Elemente mit den Klassen "info" und "wichtig"
.info .wichtig Alle Elemente die der Klasse "wichtig" angehören und sich innerhalb der Elemente mit der Klasse "info" befinden>

Attribut-Selektoren

[href] {…} Alle Elemente mit dem Attributnamen "href"
[href=http://google.de] {…} Alle Elemente deren Attribut "href" exakt den Wert "http://google.de" haben
[href*=google] {…} Alle Elemente die das Attribut "href" haben und deren Wert die Zeichenkette "google" enthalten
[href^=http] {…} Alle Elemente die das Attribut "href" haben und deren Wert mit der Zeichenkette "http" beginnen
[href$=http] {…} Alle Elemente die das Attribut "href" haben und deren Wert mit der Zeichenkette "http" enden

Block- vs. Inline-Elemente

Block-Elemente…

  • sind so breit wie ihr Elternelement, falls kein anderer Wert angegeben wird
  • sind so hoch wie ihr Inhalt, falls kein andererr Wert angegeben wird
  • können Innen- und und Außenabstände haben
  • erzeugen einen Umbruch
  • sind u.a. <p>, <div>, <form>, <header>, <nav>, <ul>, <li>, and <h1>

Inline-Elemente…

  • ignorieren Höhen- und Breitenangaben
  • befinden sich im Textfluss
  • können Abstände nach oben und unten, jedoch nicht nach links und rechts, zugewiesen werden
  • erzeugen keinen Umbruch
  • sind u.a. <a>, <span>, <b>, <em>, <i>, <cite>, <mark>, and <code>

Das Box-Modell

  • Für Block-Elemente können Höhe und Breite beliebig festgelegt weren
  • Bei Inline-Elementen werden die Maße durch den Inhalt vorgegeben
  • über die Eigenschaft display: block kann ein Inline-Element als Block-Element dargestellt werden
  • Hintergrundbilder- und farben werden in den Bereich des Inhalts, Innenabstands und Rahmen gezeichnet.
  • Die Gesamtbreite eines Elements errechnet sich durch: 
    margin-left + border-left + padding-left + width + padding-right + border-right + margin-right
  • Die Gesamthöhe eines Elements sich durch: 
    margin-top + border-top + padding-top + height + padding-bottom + border-bottom + margin-bottom

HTTP-Statuscodes

Die Antwort- oder Statuscodes von HTTP bestimmen direkt die Bedeutung der Antwort auf eine HTTP-Anfrage. Sie werden stets in der ersten Zeile der Response (= Server-Antwort) in der Form

HTTP/1.1 <Statuscode>

übermittelt, wobei der Statuscode aus einer dreistelligen Zahl und einer kurzen Beschreibung besteht. Die Statuscodes werden nach ihrer ersten Ziffer in Gruppen von unterschiedlicher Bedeutung eingeteilt.

Quelle: http://wiki.selfhtml.org/wiki/Referenz:HTTP/Antwortcodes (04.12.2012)

HTTP-Statuscodes

  • 100+ Informationen
  • 200+ Erfolg
  • 300+ Umleitung
  • 400+ Fehler beim Client
  • 500+ Fehler beim Server
    • Beispiel für eine 404 Seite

HTML - Hypertext Markup Language

Die Hypertext Markup Language (deutsch: Hypertext-Auszeichnungssprache) ist eine textbasierte Auszeichnungssprache zur Strukturierung von Inhalten wie Texten, Bildern und Hyperlinks in Dokumenten. HTML-Dokumente sind die Grundlage des World Wide Web und werden von einem Webbrowser dargestellt.

Quelle: http://de.wikipedia.org/wiki/Hypertext_Markup_Language (04.12.2012)

Zeichencodierung, Sonderzeichen & Co.

Eine Zeichenkodierung (englisch Character encoding, kurz Encoding) erlaubt die eindeutige Zuordnung von Schriftzeichen (Buchstaben oder Ziffern) und Symbolen innerhalb eines Zeichensatzes. In der Elektronischen Datenverarbeitung werden Zeichen über einen Zahlenwert kodiert und eignen sich damit für Übertragungen oder Speicherungen.

Quelle: http://de.wikipedia.org/wiki/Zeichensatz (23.11.2013)

Zeichencodierung, Sonderzeichen & Co.

Zeichenkodierung für ein HTML-Dokument festlegen:

                        
<head>
    <meta http-equiv="content-type" content="text/html; charset=ISO-8859-1">
</head>

Wird in HTML5 ersetzt durch:

                        
<head>
